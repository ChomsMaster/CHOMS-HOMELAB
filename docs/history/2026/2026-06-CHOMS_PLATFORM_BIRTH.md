# CHOMS Platform

# Engineering Journal

## Project Birth

**Date:** June 2026

**Author:** Oscar Manuel Salcedo Chirinos

---

# Introduction

Every engineering project has a beginning.

Some start because a company needs a solution.

Others begin because a customer requests a product.

CHOMS Platform began for a completely different reason.

It was born from frustration.

---

# The Context

After spending a significant period away from the technology industry, finding opportunities to return became increasingly difficult.

Despite years of professional experience in software development, banking systems, infrastructure, system analysis and technical leadership, the employment market often judged experience based only on recent positions.

The longer this situation continued, the stronger the feeling became that existing knowledge was no longer visible.

Rather than accepting that perception, a decision was made.

Instead of explaining previous experience, it would be demonstrated.

---

# The Original Objective

The first objective was relatively modest.

Build a technical project capable of demonstrating engineering knowledge to future employers.

The project would become part of a professional portfolio hosted on GitHub.

Initially, the idea focused on developing ShiftCore, an orchestration framework intended to automate operational workflows.

However, another practical problem soon emerged.

---

# The Hosting Problem

Maintaining a traditional hosting provider had become unnecessarily expensive.

Rather than continuing to depend on external infrastructure, a different question appeared.

Why not build the infrastructure instead?

That question changed the direction of the entire project.

Instead of purchasing services, the objective became engineering them.

---

# The First Infrastructure Vision

The first vision was intentionally simple.

A small server capable of hosting:

* Personal websites.
* VPN services.
* Databases.
* Docker containers.

At that moment, the project was still considered a traditional homelab.

Nothing suggested that it would evolve beyond that scope.

---

# The Turning Point

As architectural discussions progressed, an important realisation emerged.

If the platform was going to be built, it should be built correctly.

Not quickly.

Not minimally.

Not simply to demonstrate Docker containers.

Instead, it should follow the same engineering discipline expected inside professional organisations.

That single decision fundamentally changed the project's direction.

---

# From Homelab to Platform

The project stopped being viewed as a home server.

Instead, it became an engineering platform.

Every component would be analysed before implementation.

Every decision would be documented.

Every architectural choice would have justification.

Every phase would prepare the next.

The infrastructure itself became only one part of a much larger engineering exercise.

---

# A Living Engineering Project

CHOMS Platform is intentionally designed to evolve over many years.

The objective is not simply to deploy services.

The objective is to demonstrate disciplined engineering.

Every document.

Every decision.

Every architecture diagram.

Every deployment.

Every success.

Every mistake.

Becomes part of the engineering history of the platform.

---

# Looking Forward

At the time this journal entry was written, the platform remained in its earliest architectural stages.

Most services had yet to be deployed.

Much of the documentation was still under development.

The future architecture already envisioned dedicated storage, multiple compute nodes, automation, monitoring, security, disaster recovery, Kubernetes and artificial intelligence workloads.

The journey had only just begun.

---

# Reflection

This project was never created to prove that technology works.

Technology already works.

CHOMS Platform exists to demonstrate how technology can be engineered with discipline, structure, documentation and long-term vision.

Regardless of how large the platform eventually becomes, this moment will always represent its true beginning.

---

*"Every platform starts with hardware.*

*CHOMS Platform started with a decision."*
