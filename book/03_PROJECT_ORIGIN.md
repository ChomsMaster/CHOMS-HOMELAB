# Project Origin

## Every project begins with a single idea

Looking back today, it is tempting to believe that CHOMS Platform was conceived as a self-hosted infrastructure platform from the very beginning.

That is not true.

The first idea had nothing to do with servers.

It had nothing to do with Docker, Kubernetes, clusters or private cloud infrastructure.

The original idea was ShiftCore.

ShiftCore was envisioned as a software platform capable of orchestrating repetitive operational processes across multiple services and platforms.

It represented the long-term software vision.

At that point there was no intention of building an entire infrastructure.

Infrastructure was simply assumed to exist somewhere else.

---

## The VPN came first

While thinking about ShiftCore, another practical problem appeared.

Living in Spain while maintaining personal and professional connections with Venezuela created an everyday inconvenience.

Many Venezuelan online services restricted access from outside the country.

At the same time, certain Spanish services were not always easily accessible when travelling back to Venezuela.

The obvious solution was to build private VPN infrastructure.

One VPN endpoint would remain in Venezuela.

Another would operate in Spain.

Together they would provide secure connectivity regardless of physical location.

The first VPN was successfully deployed.

Although simple, it introduced an important realization.

Owning infrastructure provides freedom.

Instead of depending entirely on external providers, services could begin moving under direct control.

This was the first turning point of the project.

---

## Losing the hosting provider

Around the same period another event accelerated the transformation.

For financial reasons it became impossible to continue paying the commercial hosting provider used for the ChomsMaster website.

Rather than seeing that situation as a limitation, it became an opportunity.

A simple question emerged.

Why continue renting infrastructure if it could be built?

Initially the answer seemed obvious.

Purchase a small computer.

Install Linux.

Host the website.

Deploy the VPN.

Problem solved.

That was the original scope.

Or so it seemed.

---

## The Homelab idea

Research quickly introduced the concept of the Homelab.

Small personal infrastructures built using inexpensive hardware.

Most of the examples found online focused on experimentation.

Install a few containers.

Deploy a NAS.

Run Plex.

Learn Docker.

Interesting projects.

Useful projects.

But none of them reflected the engineering vision that was forming.

The objective was never to own a server.

The objective was to build an engineering platform.

Infrastructure was simply the foundation.

---

## A change in mindset

At this point the project changed completely.

Instead of asking:

"Which services should I install?"

A different question appeared.

"How would a professional engineering organisation build this platform if it started from zero?"

That question changed every subsequent decision.

Documentation became mandatory.

Version control became mandatory.

Architecture reviews became mandatory.

Operational procedures became mandatory.

Every change would leave an audit trail.

Every architectural decision would be recorded.

Every phase would have defined objectives and completion criteria.

The project stopped behaving like a personal laboratory.

It started behaving like an enterprise engineering programme.

---

## Beyond the Homelab

As new ideas appeared, the scope expanded naturally.

The infrastructure would eventually support much more than personal services.

Among the long-term objectives are:

- The ChomsMaster public website.
- ShiftCore backend services and APIs.
- Personal private cloud infrastructure.
- Monitoring and observability.
- Secure remote access.
- Development environments.
- CI/CD pipelines.
- Future Kubernetes clusters.
- Educational platforms for schools.
- Backup infrastructure.
- Disaster recovery capabilities.
- AI experimentation environments.
- Future distributed computing nodes.

The Homelab was no longer the destination.

It became the starting point.

---

## The platform vision

Today the project is intentionally referred to as CHOMS Platform rather than simply CHOMS-HOMELAB.

The Homelab represents only the first implementation.

The platform itself is expected to evolve continuously over many years.

Hardware will change.

Technologies will evolve.

Services will be replaced.

Entire architectures may disappear.

The engineering principles, however, should remain constant.

This repository is therefore not intended to document a server.

It documents the evolution of an engineering platform from its very first idea.

And that evolution has only just begun.
