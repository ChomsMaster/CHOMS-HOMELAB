# CHOMS-HOMELAB Case Study

## Background

CHOMS-HOMELAB started as a practical solution after recurring hosting costs became difficult to justify.

Instead of moving to another hosting provider, the decision was made to design and build a complete self-hosted infrastructure using a low-cost Mini PC.

The project gradually evolved into a long-term engineering platform used for learning, experimentation and production-inspired deployments.

---

## Objectives

- Replace external hosting services.
- Learn Linux system administration.
- Build production-like infrastructure.
- Improve backend and DevOps skills.
- Create a professional engineering portfolio.

---

## Technical Stack

- Debian 13
- Docker
- Docker Compose
- WireGuard
- Pi-hole
- PostgreSQL 17
- Nginx
- Git
- GitHub

---

## Current Status

The infrastructure currently provides:

- Secure VPN access
- DNS filtering
- Containerized services
- PostgreSQL database
- Operational scripts
- Technical documentation
- Architecture diagrams

Future work includes:

- FastAPI
- Nextcloud
- Monitoring
- Mail Server
- CHOMS Dashboard
- CHOMS Master API
- ShiftCore integration

---

## Lessons

The project demonstrates that a production-inspired infrastructure can be built progressively using low-cost hardware while maintaining engineering standards such as documentation, version control and reproducibility.
